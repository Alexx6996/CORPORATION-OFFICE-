# Network Baseline (CORPORATION / OFFICE)

## Цель
Зафиксировать сетевую модель доступа к backend ядру CORPORATION / OFFICE и исключить несанкционированную публикацию сервиса в сеть.

## Модель доступа
1. Backend ядро CORPORATION / OFFICE выполняется как Windows-служба `AIOFFICESSvc`.
2. Приложение (uvicorn FastAPI) слушает только на `127.0.0.1:8181` (loopback).
   - Это означает: сервис доступен только локально на этом хосте.
   - Внешний хост в сети не может подключиться напрямую.
3. Порт 8181 не объявляется в Windows Firewall для входящих соединений.
   - Проверка выполняется через аудит inbound-правил (`Get-NetFirewallRule` + `Get-NetFirewallPortFilter` для порта 8181).
   - Любое правило, разрешающее Inbound на 8181 извне, считается нарушением политики безопасности.

## Требования эксплуатации
1. Служба `AIOFFICESSvc` не публикуется наружу. Прямой сетевой доступ к uvicorn извне запрещён.
2. Любой внешний клиент (UI, панель, мобильный агент, удалённый пользователь) должен ходить не напрямую в uvicorn, а через доверенный фронт:
   - reverse proxy с аутентификацией и авторизацией (конкретная реализация TBD),
   - контроль доступа и журналирование обращений.
3. Любая попытка пробросить порт 8181 (локальный loopback) наружу без утверждённого reverse proxy считается отклонением от базовой модели сети и требует отдельного ADR.

## Значение для аудита / безопасности
- Это исключает "теневые" публикации backend в LAN/интернет.
- Это фиксирует минимальную поверхность атаки: ядро доступно только локальному оператору на этом узле.
- Это создаёт основу для утверждения: "Система не предоставляет удалённый API без авторизации".

## Итог
CORPORATION / OFFICE работает как локальный защищённый офис-ядро.
Доступ к API бэкенда разрешён только через localhost на этом компьютере.
Любая внешняя интеграция должна идти через утверждённый прокси-слой, а не напрямую.
